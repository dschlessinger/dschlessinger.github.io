Technical Blog: Week 3

JavaScript in Web Development

August 2, 2014


Week 3 was a brief introduction to JavaScript. Although I feel like I understand the concepts in this section, I am not very confident about how to integrate them with my working knowledge of HTML and CSS. That said, after doing a little bit of web research, I'm starting to undertstand how JavaScript fits in and why it is so important to web developers. Here are a few reasons:

  1. JavaScript started on Netscape Navigator. Hopefully, you never had to experience this terrible browser, but it really was one of the most used web tools in the 1990s. As a result, JavaScript gained a stronghold as a browser-oriented programming language from early on in the development of the web.
  2. JavaScript is fast and flexible. As the name would suggest, everything is a script. This means that the language doesn't need an interpreter or compiler to run.
  3. Javascript is accessible. It is supported by virtually every browser.
  4. JavaScript makes web pages come alive.
  5. JavaScript's node.js is very powerful. Google wrote node.js into their Chrome browser as a back end extension of JavaScript, and in the process redesigned the wheel. They gave JavaScript asychronous, back end processing abilities, which allows it to process multiple tasks at once.

JavaScript does have some limitations. For example, potential security issues exist because of snippets, which are bits of code that execute immediately on a webpage. If hacked, they could be used maliciously. Overall, though, JavaScript is an incredibly powerful language, and one that will almost certainly remain that way for a while.
