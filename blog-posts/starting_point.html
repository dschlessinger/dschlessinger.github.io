Responsive Design and Fluid Layouts

An up-and-coming way to resize webpages to fit any screen resolution

By: Daniel Schlessinger and Bryan Woodhams

Problem that Responsive Design Fixes

Over the course of our daily lives we encounter many different screens. We use our laptops to read the news, our smart phones to stay current later in the day. When you change between these different resolutions, websites look entirely different. This difference in appearance comes from serveral areas. The goal of responsive design is to create a website that uses stylesheets stragetically, either with percentage widths or conditional formatting, to fit many different browsers and devices.

How Responsive Design Works

In the past, web designers have laid out web pages in terms of fixed pixels. This works fine with most standard browsers, but smaller devices pose problems. According to an August 2012 blog post on Treehouse, one way to get around this issue is to convert the pixel widths of various elements into proportional widths. In this case, the original width in pixels is divided by the total number of pixels on a standard screen to determine the percentage of the screen that it occupies. Then, if a user resizes their screen or accesses the page on a smaller device, the code automatically recalculates the width of an element to fit the new resolution.

Takeaways of Responsive Design

Given the current paths of the web and the multitude of devices that are accessing the it, responsive design looks to be the future of web design. The ability to naturally fit many different screens leads to a reduction in work required to have the most user friendly experience. We have covered only one approach to responsive design. There are many more ways to achieve this design philosophy using HTML and CSS creatively. If you want more information on these approaches to responsive design, be sure to check out this awesome article from Smashing Magazine. Good luck!
